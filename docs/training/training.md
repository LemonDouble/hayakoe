# ② 전처리 & 학습

[① 데이터 준비](/training/data-prep) 에서 만든 데이터셋을 학습용으로 가공하고, 목소리를 학습합니다.

학습은 컴퓨터 자원을 굉장히 많이 쓰는 장기간 작업입니다. GUI 또한 컴퓨터 자원을 소모하기 때문에, 꼭 UI 가 필요한 데이터 전처리 파이프라인 이후는 컴퓨터 자원을 거의 소모하지 않는 CLI 로 분리해 두었습니다.

명령어를 외울 필요는 없습니다. CLI 를 실행한 뒤 메뉴를 따라가면 됩니다.

## CLI 실행

레포 루트에서 인자 없이 실행하면 메인 메뉴가 뜹니다.

```bash
uv run poe cli
```

## 전처리

데이터 준비에서 만든 `(음성, 텍스트)` 쌍은 모델이 바로 학습하기엔 맞지 않는 형태입니다. 이 원본을 **모델이 이해할 수 있는 숫자 형태로 변환해 캐시해 두는 단계** 가 전처리입니다.

데이터셋을 고르면 현재 전처리 상태 패널이 먼저 뜹니다.

```text
╭──────────────────────────────── 츠쿠요미 — 전처리 상태 ────────────────────────────────╮
│    ✓  오디오 파일        72개                                                          │
│    ○  텍스트 전처리      미완료                                                        │
│    ○  BERT 임베딩        0/72                                                          │
│    ○  스타일 벡터        0/72                                                          │
│    ○  기본 스타일        미완료                                                        │
╰────────────────────────────────────────────────────────────────────────────────────────╯
다음 단계: 전처리 실행으로 데이터를 준비하세요.

? 츠쿠요미 — 무엇을 할까요?
  ❯ 전처리 실행 (남은 단계)
    전처리 전체 재실행
    학습 설정 편집
    뒤로
```

**전처리 실행 (남은 단계)** 를 선택하면 학습에 필요한 4단계 전처리가 차례로 돕니다.

완료 판정은 자동으로 되므로, 중간에 끊기고 다시 들어와도 끝난 단계는 스킵합니다. 전부 다시 돌리고 싶으면 **전처리 전체 재실행** 을 쓰세요.

::: tip G2P 에러가 났다면
전처리 1단계에서 `text_error.log` 가 생겼다면, 일부 문장이 음소 변환에 실패했다는 뜻입니다.

해당 줄은 학습 대상에서 자동으로 빠지지만, 데이터가 적은 상황이라면 로그를 열어 원인을 확인해 보세요. 특수 기호나 읽을 수 없는 한자가 원인인 경우가 많습니다.
:::

::: details 내부 동작 — 4단계 상세
| 단계 | 하는 일 | 산출물 |
|---|---|---|
| 1. G2P 텍스트 전처리 | 일본어 텍스트를 발음(음소 + 억양 톤)으로 변환. 모델은 글자가 아니라 **발음** 을 학습합니다. | `train.list` / `val.list` 가 7필드로 확장, `text_error.log` |
| 2. BERT 임베딩 | 문장 맥락을 DeBERTa 임베딩으로 미리 계산해 캐시. 자연스러운 억양을 위해 참고하는 값인데, 학습 스텝마다 BERT 를 돌리면 느리므로 한 번만 계산합니다. | `<wav>.bert.pt` |
| 3. 스타일 벡터 | 각 음성에서 화자의 목소리 특징(톤·말투)을 벡터로 추출 (pyannote 임베딩). | `<wav>.npy` (NaN 파일 자동 제거) |
| 4. 기본 스타일 | 위 벡터들의 평균을 화자의 "표준 스타일" 로 저장. | `exports/<model>/style_vectors.npy`, `config.json` |

완료 판정은 파일 개수 매칭으로 합니다.
:::

## 학습 설정 편집

전처리가 끝나면 **학습 설정 편집** 으로 주요 학습 파라미터를 고칠 수 있습니다.

수십 개 필드 전부가 아니라 **실전에서 의미 있는 11개만** 골라두었습니다.

```text
╭────────────────────────────────────── 학습 설정 ───────────────────────────────────────╮
│  ┏━━━━━━━━━━━━━━━━━━━━━━━━━━━┳━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┳━━━━━━━━━━━━┓  │
│  ┃  파라미터                 ┃  설명                                   ┃        값  ┃  │
│  ┡━━━━━━━━━━━━━━━━━━━━━━━━━━━╇━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━╇━━━━━━━━━━━━┩  │
│  │  기본 설정                │                                         │            │  │
│  │    epochs                 │  학습 에포크 수                         │       500  │  │
│  │    batch_size             │  배치 크기 (GPU당)                      │         2  │  │
│  │    nproc_per_node         │  GPU 수                                 │         1  │  │
│  │    bf16_run               │  bfloat16 혼합 정밀도                   │       OFF  │  │
│  │    eval_interval          │  체크포인트 저장 간격 (스텝)            │      1000  │  │
│  │    log_interval           │  텐서보드 로깅 간격 (스텝)              │       200  │  │
│  │                           │                                         │            │  │
│  │  고급 설정                │                                         │            │  │
│  │    learning_rate          │  학습률                                 │    0.0001  │  │
│  │    warmup_epochs          │  워밍업 에포크                          │         0  │  │
│  │    freeze_JP_bert         │  JP BERT 인코더 동결                    │       OFF  │  │
│  │    freeze_style           │  스타일 인코더 동결                     │       OFF  │  │
│  │    freeze_decoder         │  디코더 동결                            │       OFF  │  │
│  └───────────────────────────┴─────────────────────────────────────────┴────────────┘  │
╰────────────────────────────────────────────────────────────────────────────────────────╯
```

::: tip 총 스텝 2000~4000 이 보통 기준
경험상 **총 스텝 2000~4000** 구간에서 쓸 만한 음성이 나오는 경우가 많습니다. 에포크 수는 데이터셋 크기와 배치 크기에 따라 달라지므로, 브리핑 패널의 `총 스텝` 이 이 범위에 들어오도록 `epochs` 를 맞추세요.

이 범위 안에서 체크포인트가 여러 개 나오도록 `eval_interval` 을 500~1000 정도로 잡으면, 나중에 [품질 리포트](/training/quality-check) 에서 가장 좋은 시점을 고를 때 편합니다.
:::

각 파라미터의 의미와 권장값(높이면 / 낮추면 / 권장)은 접어 두었습니다. 조정이 필요할 때 펼쳐 보세요.

::: details 파라미터별 상세 — 기본 설정 (6개)
#### `train.epochs` — 학습 에포크 수
전체 학습 데이터를 처음부터 끝까지 몇 번 반복 학습할지 설정합니다.

- **높이면**: 모델이 더 오래 학습하여 음질이 향상될 수 있습니다. 다만 너무 높으면 학습 데이터를 통째로 외워버려서, 새로운 문장을 읽을 때 오히려 부자연스러워집니다 (과적합).
- **낮추면**: 학습이 빨리 끝나지만, 모델이 충분히 배우지 못해 음질이 낮을 수 있습니다.
- **권장**: 총 스텝이 2000~4000 에 들어오도록 맞추세요. `에포크 = 목표 스텝 × 배치 크기 ÷ 학습 데이터 수` 로 계산하면 됩니다.

#### `train.batch_size` — 배치 크기 (GPU당)
한 번에 몇 개의 음성 샘플을 묶어서 학습할지 설정합니다.

- **높이면**: 학습 속도가 빨라지고 안정적이지만, GPU 메모리(VRAM)를 더 많이 사용합니다. 메모리가 부족하면 `CUDA out of memory` 에러가 발생합니다.
- **낮추면**: 적은 메모리로도 학습 가능하지만, 속도가 느려집니다.
- **권장**: RTX 3060 (12GB) → 2~4, RTX 3090/4090 (24GB) → 4~8.

#### `train.nproc_per_node` — GPU 수
학습에 사용할 GPU 수입니다. 여러 GPU가 있으면 데이터를 나누어 병렬로 학습하여 속도가 비례해서 빨라집니다 (DistributedDataParallel).

- **기본값 1**: 단일 GPU 학습. GPU가 하나면 이 값을 바꾸지 마세요.
- 실제 장착된 수를 초과하면 에러가 납니다. `nvidia-smi` 로 확인할 수 있습니다.

#### `train.bf16_run` — bfloat16 혼합 정밀도
계산 정밀도를 32비트에서 16비트로 줄여 학습 속도를 높이는 기법입니다.

- **켜면**: 학습 속도 ~1.5배, VRAM 사용량 감소. 단 Ampere 이상 GPU(RTX 30xx, 40xx, A100 등)가 필요합니다.
- **끄면**: 모든 GPU에서 동작하지만 더 느립니다.
- **주의**: RTX 20xx 이하에서 켜면 에러가 나거나 품질이 떨어질 수 있습니다.

#### `train.eval_interval` — 체크포인트 저장 간격 (스텝)
몇 스텝마다 체크포인트를 저장할지 설정합니다. 체크포인트는 학습 중 모델의 스냅샷으로, 나중에 가장 좋은 시점의 모델을 골라 쓸 수 있게 해줍니다.

- **자주 저장**: 세밀하게 비교 가능, 대신 디스크 사용량이 큽니다 (체크포인트당 ~240MB).
- **드물게 저장**: 디스크는 절약되지만 좋은 시점을 놓칠 수 있습니다.
- **권장**: 500~1000 스텝. 총 스텝(2000~4000) 구간에서 체크포인트가 4~8개 남도록 잡으면 품질 리포트에서 비교할 때 편합니다.

#### `train.log_interval` — 텐서보드 로깅 간격 (스텝)
몇 스텝마다 loss 같은 지표를 TensorBoard에 기록할지 설정합니다.

- **권장**: 대부분의 경우 기본값(200)이면 충분합니다.

:::

::: details 파라미터별 상세 — 고급 설정 (5개)
#### `train.learning_rate` — 학습률
모델이 한 스텝마다 얼마나 크게 수정될지 결정합니다. 자전거 핸들의 민감도와 비슷합니다.

- **높이면**: 빠르게 학습하지만 값이 튀어 학습이 불안정해지거나 발산할 수 있습니다.
- **낮추면**: 안정적이지만 학습이 매우 느리고 최적점에 도달하지 못할 수 있습니다.
- **권장**: 잘 모르겠으면 기본값(0.0001)을 그대로 쓰세요.

#### `train.warmup_epochs` — 워밍업 에포크
학습 초반에 학습률을 0부터 서서히 올리는 구간입니다. 자동차를 출발할 때 급가속 대신 천천히 속도를 올리는 것과 비슷합니다.

- **권장**: 0~3 에포크. 학습이 초반에 불안정하면 1~2로 설정해보세요.

#### `train.freeze_JP_bert` — JP BERT 인코더 동결
텍스트를 이해하는 부분(BERT)의 가중치를 고정합니다.

- **켜면**: BERT가 이미 학습한 일본어 지식을 보존합니다. 데이터가 적을 때(~200문장 이하) 과적합 방지에 효과적이고, 속도와 메모리도 개선됩니다.
- **끄면**: BERT도 함께 미세조정되어, 데이터가 충분할 때 더 자연스러운 발음과 억양을 학습합니다.
- **권장**: 200문장 이하 → ON, 500문장 이상 → OFF.

#### `train.freeze_style` — 스타일 인코더 동결
화자의 말투/감정을 표현하는 스타일 인코더의 가중치를 고정합니다.

- **켜면**: 기존 스타일 표현력을 보존합니다. 데이터가 적거나 기존 감정 표현력을 유지하고 싶을 때 유용합니다.
- **끄면**: 새로운 화자의 스타일 특성을 자유롭게 학습합니다.
- **권장**: 일반적으로 OFF. 학습 결과가 단조롭다면 ON으로 시도해보세요.

#### `train.freeze_decoder` — 디코더 동결
최종 음성을 생성하는 디코더의 가중치를 고정합니다.

- **켜면**: 음성 생성 부분은 변경되지 않고, 텍스트 해석 부분만 학습됩니다. 학습이 빨라지지만 음질 개선 폭이 제한됩니다.
- **끄면**: 디코더도 함께 학습하여 해당 화자에 맞는 음질을 낼 수 있습니다.
- **권장**: 일반적으로 OFF. 특수한 경우가 아니면 끌 필요 없습니다.
:::

## 학습 시작

**학습 시작** 을 누르면 **브리핑 패널** 이 먼저 뜹니다. 에포크·배치 크기·총 스텝·체크포인트 간격·예상 디스크 사용량이 한 표에 정리되어 나오고, 체크포인트 간격만 그 자리에서 바로 수정할 수 있습니다.

```text
╭──────────────────────────────── 츠쿠요미 — 학습 브리핑 ────────────────────────────────╮
│    하이퍼파라미터                                                                      │
│      데이터셋                            65 train / 7 val                              │
│      에포크                                           500                              │
│      배치 크기                                          2                              │
│      학습률                                        0.0001                              │
│      bfloat16                                         OFF                              │
│                                                                                        │
│    스텝 & 체크포인트                                                                   │
│      에포크당 스텝                                     33                              │
│      총 스텝                                       16,500                              │
│      로그 간격                                  200 steps                              │
│      체크포인트 간격                           1000 steps                              │
│      예상 저장 횟수                                  16회                              │
│      예상 디스크 사용량             ~3.8GB (16개 × 240MB)                              │
│                                                                                        │
│    저장 경로                                                                           │
│      체크포인트            data/dataset/츠쿠요미/training                              │
│      추론 모델              data/dataset/츠쿠요미/exports                              │
╰────────────────────────────────────────────────────────────────────────────────────────╯

? 진행
  ❯ 학습 시작
    체크포인트 간격 변경 (현재: 1000 steps)
    취소
```

확인하고 진행하면 학습이 시작되고, TensorBoard 도 자동으로 같이 뜹니다. 콘솔에 찍힌 URL 을 브라우저로 열면 loss 변화를 실시간으로 볼 수 있습니다.

::: tip TensorBoard 는 알아서 켜집니다
별도로 `tensorboard --logdir ...` 을 칠 필요가 없습니다.

학습이 끝나거나 `Ctrl-C` 로 중단하면 자동으로 닫힙니다.
:::

::: details 내부 동작 — 학습 시작 시 일어나는 일
1. **사전학습 모델 자동 다운로드** — `hf://lemondouble/hayakoe/pretrained/<name>` 에서 `G_0.safetensors`, `D_0.safetensors`, `WD_0.safetensors` 를 캐시에 받습니다. 이미 `G_*.pth` 체크포인트가 있으면 "재개 모드" 로 판단해 스킵합니다.
2. **학습 프로세스 기동** — `torch.distributed.run` 서브프로세스로 `train_ms_jp_extra.py` (SBV2 원본 포팅) 를 띄웁니다. `nproc_per_node` 값이 그대로 넘어갑니다.
3. **TensorBoard 자동 실행** — 학습 디렉토리를 바라보는 TensorBoard 가 같이 뜹니다.

예상 디스크 사용량은 체크포인트당 240MB 고정 추정입니다.
:::

## 학습 결과물

학습이 끝나면 데이터셋 디렉토리 아래에 두 경로가 채워집니다.

- **`training/`** — 학습 중간 스냅샷 (체크포인트·TensorBoard 로그). 디스크를 많이 먹으니, 학습 후 필요 없는 체크포인트는 지워도 됩니다.
- **`exports/<model_name>/`** — 배포와 평가의 기준이 되는 최종 모델. 다음 단계 ([③ 품질 리포트](/training/quality-check), [④ 배포](/training/publish)) 가 이 디렉토리를 읽습니다.

::: details 내부 동작 — 디렉토리 구조
```
<dataset_root>/
├── training/                          # 학습 중간 산출물
│   ├── G_*.pth, D_*.pth, WD_*.pth    # 체크포인트 (eval_interval마다)
│   └── events.out.tfevents.*          # TensorBoard 로그
└── exports/<model_name>/              # 최종 모델
    ├── *.safetensors
    ├── config.json
    └── style_vectors.npy
```
:::

## 중단과 재시작

학습을 `Ctrl-C` 로 끊어도 체크포인트는 `training/` 아래에 남습니다.

같은 데이터셋으로 다시 **학습 시작** 을 누르면 남아있는 체크포인트를 자동으로 감지해 재개 모드로 들어갑니다. 사전학습 다운로드는 이 경우 스킵됩니다.
