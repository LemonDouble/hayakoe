# ② 前處理 & 訓練

將 [① 資料準備](/zh-TW/training/data-prep) 中製作的資料集加工為訓練用格式,並訓練聲音。

訓練是極其消耗計算資源的長時間任務。GUI 同樣消耗計算資源,因此在必須使用 UI 的資料前處理流水線之後,分離為幾乎不消耗計算資源的 CLI。

不需要記住指令。啟動 CLI 後按選單操作即可。

在儲存庫根目錄不帶參數執行,即可看到主選單。

## 前處理

資料準備中製作的 `(語音, 文字)` 對的形式並不適合模型直接訓練。**將原始資料轉換為模型可理解的數值形式並快取** 的步驟就是前處理。

選擇資料集後會先顯示目前前處理狀態面板。

```text
╭──────────────────────────────── tsukuyomi — 前處理狀態 ────────────────────────────────╮
│    ✓  音訊檔案        72個                                                             │
│    ○  文字前處理      未完成                                                           │
│    ○  BERT 嵌入       0/72                                                             │
│    ○  風格向量        0/72                                                             │
│    ○  預設風格        未完成                                                           │
╰────────────────────────────────────────────────────────────────────────────────────────╯
下一步: 執行前處理以準備資料。

? tsukuyomi — 要做什麼?
  ❯ 執行前處理 (剩餘步驟)
    重新執行全部前處理
    編輯訓練設定
    返回
```

選擇 **執行前處理 (剩餘步驟)** 後訓練所需的 4 步前處理會依序執行。

完成判定是自動的,中途中斷再次進入時已完成的步驟會被跳過。想全部重新執行請使用 **重新執行全部前處理**。

::: tip 如果 G2P 出錯
如果前處理第 1 步生成了 `text_error.log`,表示部分句子音素轉換失敗。

這些行會自動從訓練對象中排除,但如果資料較少,建議打開日誌查看原因。通常是特殊符號或無法識別的漢字導致的。
:::

::: details 內部機制 — 4 步詳情
| 步驟 | 工作內容 | 產出 |
|---|---|---|
| 1. G2P 文字前處理 | 將日語文字轉換為發音(音素 + 聲調)。模型學習的是 **發音** 而非文字。 | `train.list` / `val.list` 擴展為 7 欄位, `text_error.log` |
| 2. BERT 嵌入 | 用 DeBERTa 嵌入預先計算句子上下文並快取。這是合成自然韻律時參考的值,由於每個訓練步驟都執行 BERT 太慢,所以只計算一次。 | `<wav>.bert.pt` |
| 3. 風格向量 | 從每段語音中擷取話者的聲音特徵(音色·說話方式)為向量(pyannote 嵌入)。 | `<wav>.npy` (NaN 檔案自動刪除) |
| 4. 預設風格 | 將上述向量的平均值儲存為話者的「標準風格」。 | `exports/<model>/style_vectors.npy`, `config.json` |

完成判定透過檔案數量比對進行。
:::

## 編輯訓練設定

前處理完成後可以透過 **編輯訓練設定** 修改主要訓練參數。

::: tip 總步數 2000~4000 是一般標準
經驗上 **總步數 2000~4000** 區間內往往能產出可用的語音。輪數因資料集大小與批大小而異,請調整 `epochs` 使簡報面板中的 `總步數` 落在此範圍內。

在此範圍內設定 `eval_interval` 為 500~1000 使多個檢查點產出,之後在 [品質報告](/zh-TW/training/quality-check) 中選擇最佳時間點時會更方便。
:::

各參數的含義與建議值(提高/降低/建議)已折疊。需要調整時展開查看。

::: details 各參數詳情 — 基本設定 (6 個)
#### `train.epochs` — 訓練輪數
設定將全部訓練資料從頭到尾重複學習多少遍。

- **提高**: 模型訓練時間更長,音質可能提升。但過高會導致模型完全記住訓練資料,朗讀新句子時反而不自然(過擬合)。
- **降低**: 訓練更快完成,但模型可能學習不充分,音質較低。
- **建議**: 調整使總步數落在 2000~4000 之間。`輪數 = 目標步數 × 批大小 ÷ 訓練資料數` 計算即可。

#### `train.batch_size` — 批大小 (每 GPU)
設定每次將多少個語音範例打包一起訓練。

- **提高**: 訓練速度加快且更穩定,但 GPU 顯示記憶體(VRAM)佔用增加。記憶體不足時會出現 `CUDA out of memory` 錯誤。
- **降低**: 可以用更少記憶體訓練,但速度變慢。
- **建議**: RTX 3060 (12GB) → 2~4, RTX 3090/4090 (24GB) → 4~8。

#### `train.nproc_per_node` — GPU 數
用於訓練的 GPU 數量。多 GPU 時資料分散平行訓練,速度按比例提升 (DistributedDataParallel)。

- **預設值 1**: 單 GPU 訓練。只有一塊 GPU 時不要修改此值。
- 超過實際安裝數量會發生錯誤。可用 `nvidia-smi` 確認。

#### `train.bf16_run` — bfloat16 混合精度
將計算精度從 32 位降至 16 位以提高訓練速度的技術。

- **開啟**: 訓練速度提升約 1.5 倍,VRAM 使用減少。但需要 Ampere 及以上 GPU(RTX 30xx, 40xx, A100 等)。
- **關閉**: 所有 GPU 都能執行但速度更慢。
- **注意**: 在 RTX 20xx 及以下開啟可能會發生錯誤或降低品質。

#### `train.eval_interval` — 檢查點儲存間隔(步)
設定每多少步儲存一次檢查點。檢查點是訓練中模型的快照,之後可以選出最佳時間點的模型使用。

- **頻繁儲存**: 可以細粒度比較,但磁碟佔用大(每個檢查點約 240MB)。
- **稀少儲存**: 節省磁碟但可能錯過好的時間點。
- **建議**: 500~1000 步。在總步數(2000~4000)區間內留下 4~8 個檢查點,品質報告中比較時會更方便。

#### `train.log_interval` — TensorBoard 記錄間隔(步)
設定每多少步記錄一次 loss 等指標到 TensorBoard。

- **建議**: 大多數情況下預設值(200)即可。

:::

::: details 各參數詳情 — 進階設定 (5 個)
#### `train.learning_rate` — 學習率
決定模型每一步修正幅度的大小。類似自行車把手的靈敏度。

- **提高**: 學習更快但值可能跳動,訓練不穩定或發散。
- **降低**: 更穩定但訓練非常慢,可能無法達到最佳點。
- **建議**: 不確定的話就用預設值(0.0001)。

#### `train.warmup_epochs` — 預熱輪數
訓練初期將學習率從 0 逐漸提升的區間。類似汽車起步時不急加速而是緩慢提速。

- **建議**: 0~3 輪。如果訓練初期不穩定可嘗試設為 1~2。

#### `train.freeze_JP_bert` — 凍結 JP BERT 編碼器
固定理解文字的部分(BERT)的權重。

- **開啟**: 保留 BERT 已學習的日語知識。資料較少時(約 200 句以下)有效防止過擬合,速度與記憶體也會改善。
- **關閉**: BERT 也一起微調,資料充分時可以學到更自然的發音與韻律。
- **建議**: 200 句以下 → ON,500 句以上 → OFF。

#### `train.freeze_style` — 凍結風格編碼器
固定表達話者說話方式/情緒的風格編碼器權重。

- **開啟**: 保留現有的風格表現力。資料較少或想保持現有情感表現力時有用。
- **關閉**: 自由學習新話者的風格特徵。
- **建議**: 一般設為 OFF。如果訓練結果單調可嘗試 ON。

#### `train.freeze_decoder` — 凍結解碼器
固定最終生成語音的解碼器權重。

- **開啟**: 語音生成部分不變,僅訓練文字剖析部分。訓練更快但音質改善幅度受限。
- **關閉**: 解碼器也一起訓練,可以產出適合該話者的音質。
- **建議**: 一般設為 OFF。除非特殊情況無需開啟。
:::

## 開始訓練

點擊 **開始訓練** 後會先顯示 **簡報面板**。輪數·批大小·總步數·檢查點間隔·預計磁碟佔用整理在一張表中,檢查點間隔可以當場直接修改。

```text
╭──────────────────────────────── tsukuyomi — 訓練簡報 ────────────────────────────────╮
│    超參數                                                                             │
│      資料集                            65 train / 7 val                               │
│      輪數                                            500                              │
│      批大小                                            2                              │
│      學習率                                        0.0001                              │
│      bfloat16                                         OFF                              │
│                                                                                        │
│    步數 & 檢查點                                                                      │
│      每輪步數                                         33                              │
│      總步數                                       16,500                              │
│      記錄間隔                                  200 steps                              │
│      檢查點間隔                           1000 steps                                  │
│      預計儲存次數                                  16次                               │
│      預計磁碟佔用             ~3.8GB (16個 × 240MB)                                   │
│                                                                                        │
│    儲存路徑                                                                           │

  ❯ 開始訓練
    修改檢查點間隔 (目前: 1000 steps)
    取消
```

確認後繼續,訓練開始,TensorBoard 也會自動同時啟動。在瀏覽器中打開主控台列印的 URL 即可即時查看 loss 變化。

::: tip TensorBoard 會自動啟動
無需另外輸入 `tensorboard --logdir ...`。

訓練結束或用 `Ctrl-C` 中斷時會自動關閉。
:::

::: details 內部機制 — 訓練開始時發生的事
1. **預訓練模型自動下載** — 從 `hf://lemondouble/hayakoe/pretrained/<name>` 下載 `G_0.safetensors`, `D_0.safetensors`, `WD_0.safetensors` 到快取。如果已有 `G_*.pth` 檢查點則判斷為「恢復模式」並跳過。
2. **訓練行程啟動** — 透過 `torch.distributed.run` 子行程啟動 `train_ms_jp_extra.py`(SBV2 原始移植)。`nproc_per_node` 值直接傳遞。
3. **TensorBoard 自動執行** — 指向訓練目錄的 TensorBoard 同時啟動。

預計磁碟佔用按每個檢查點 240MB 固定估算。
:::

## 訓練產出

訓練完成後資料集目錄下會填充兩個路徑。

- **`training/`** — 訓練中間快照(檢查點·TensorBoard 日誌)。佔用磁碟較多,訓練後不需要的檢查點可以刪除。
- **`exports/<model_name>/`** — 作為部署與評估基準的最終模型。下一步([③ 品質報告](/zh-TW/training/quality-check),[④ 部署](/zh-TW/training/publish))讀取此目錄。

::: details 內部機制 — 目錄結構
```
<dataset_root>/
├── training/                          # 訓練中間產出
│   ├── G_*.pth, D_*.pth, WD_*.pth    # 檢查點 (每 eval_interval)
│   └── events.out.tfevents.*          # TensorBoard 日誌
└── exports/<model_name>/              # 最終模型
    ├── *.safetensors
    ├── config.json
    └── style_vectors.npy
```
:::

## 中斷與恢復

用 `Ctrl-C` 中斷訓練後檢查點仍保留在 `training/` 下。

用相同資料集再次點擊 **開始訓練** 時會自動偵測到殘留檢查點並進入恢復模式。此時跳過預訓練下載。
