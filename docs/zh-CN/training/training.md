# ② 预处理 & 训练

将 [① 数据准备](/zh/training/data-prep) 中制作的数据集加工为训练用格式,并训练声音。

训练是极其消耗计算资源的长时间任务。GUI 同样消耗计算资源,因此在必须使用 UI 的数据预处理流水线之后,分离为几乎不消耗计算资源的 CLI。

不需要记住命令。启动 CLI 后按菜单操作即可。

在仓库根目录不带参数运行,即可看到主菜单。

## 预处理

数据准备中制作的 `(语音, 文本)` 对的形式并不适合模型直接训练。**将原始数据转换为模型可理解的数值形式并缓存** 的步骤就是预处理。

选择数据集后会先显示当前预处理状态面板。

```text
╭──────────────────────────────── tsukuyomi — 预处理状态 ────────────────────────────────╮
│    ✓  音频文件        72个                                                             │
│    ○  文本预处理      未完成                                                           │
│    ○  BERT 嵌入       0/72                                                             │
│    ○  风格向量        0/72                                                             │
│    ○  默认风格        未完成                                                           │
╰────────────────────────────────────────────────────────────────────────────────────────╯
下一步: 执行预处理以准备数据。

? tsukuyomi — 要做什么?
  ❯ 执行预处理 (剩余步骤)
    重新执行全部预处理
    编辑训练设置
    返回
```

选择 **执行预处理 (剩余步骤)** 后训练所需的 4 步预处理会依次运行。

完成判定是自动的,中途中断再次进入时已完成的步骤会被跳过。想全部重新运行请使用 **重新执行全部预处理**。

::: tip 如果 G2P 出错
如果预处理第 1 步生成了 `text_error.log`,表示部分句子音素转换失败。

这些行会自动从训练对象中排除,但如果数据较少,建议打开日志查看原因。通常是特殊符号或无法识别的汉字导致的。
:::

::: details 内部机制 — 4 步详情
| 步骤 | 工作内容 | 产出 |
|---|---|---|
| 1. G2P 文本预处理 | 将日语文本转换为发音(音素 + 声调)。模型学习的是 **发音** 而非文字。 | `train.list` / `val.list` 扩展为 7 字段, `text_error.log` |
| 2. BERT 嵌入 | 用 DeBERTa 嵌入预先计算句子上下文并缓存。这是合成自然韵律时参考的值,由于每个训练步骤都运行 BERT 太慢,所以只计算一次。 | `<wav>.bert.pt` |
| 3. 风格向量 | 从每段语音中提取说话人的声音特征(音色·说话方式)为向量(pyannote 嵌入)。 | `<wav>.npy` (NaN 文件自动删除) |
| 4. 默认风格 | 将上述向量的平均值保存为说话人的"标准风格"。 | `exports/<model>/style_vectors.npy`, `config.json` |

完成判定通过文件数量匹配进行。
:::

::: tip 总步数 2000~4000 是一般标准
经验上 **总步数 2000~4000** 区间内往往能产出可用的语音。轮数因数据集大小和批大小而异,请调整 `epochs` 使简报面板中的 `总步数` 落在此范围内。

在此范围内设置 `eval_interval` 为 500~1000 使多个检查点产出,之后在 [质量报告](/zh/training/quality-check) 中选择最佳时间点时会更方便。
:::

各参数的含义和建议值(提高/降低/建议)已折叠。需要调整时展开查看。

::: details 各参数详情 — 基本设置 (6 个)
#### `train.epochs` — 训练轮数
设置将全部训练数据从头到尾重复学习多少遍。

- **提高**: 模型训练时间更长,音质可能提升。但过高会导致模型完全记住训练数据,朗读新句子时反而不自然(过拟合)。
- **降低**: 训练更快完成,但模型可能学习不充分,音质较低。
- **建议**: 调整使总步数落在 2000~4000 之间。`轮数 = 目标步数 × 批大小 ÷ 训练数据数` 计算即可。

#### `train.batch_size` — 批大小 (每 GPU)
设置每次将多少个语音样本打包一起训练。

- **提高**: 训练速度加快且更稳定,但 GPU 显存(VRAM)占用增加。显存不足时会出现 `CUDA out of memory` 错误。
- **降低**: 可以用更少显存训练,但速度变慢。
- **建议**: RTX 3060 (12GB) → 2~4, RTX 3090/4090 (24GB) → 4~8。

#### `train.nproc_per_node` — GPU 数
用于训练的 GPU 数量。多 GPU 时数据分散并行训练,速度按比例提升 (DistributedDataParallel)。

- **默认值 1**: 单 GPU 训练。只有一块 GPU 时不要修改此值。
- 超过实际安装数量会报错。可用 `nvidia-smi` 确认。

#### `train.bf16_run` — bfloat16 混合精度
将计算精度从 32 位降至 16 位以提高训练速度的技术。

- **开启**: 训练速度提升约 1.5 倍,VRAM 使用减少。但需要 Ampere 及以上 GPU(RTX 30xx, 40xx, A100 等)。
- **关闭**: 所有 GPU 都能运行但速度更慢。
- **注意**: 在 RTX 20xx 及以下开启可能会报错或降低品质。

#### `train.eval_interval` — 检查点保存间隔(步)
设置每多少步保存一次检查点。检查点是训练中模型的快照,之后可以选出最佳时间点的模型使用。

- **频繁保存**: 可以细粒度比较,但磁盘占用大(每个检查点约 240MB)。
- **稀少保存**: 节省磁盘但可能错过好的时间点。
- **建议**: 500~1000 步。在总步数(2000~4000)区间内留下 4~8 个检查点,质量报告中比较时会更方便。

#### `train.log_interval` — TensorBoard 记录间隔(步)
设置每多少步记录一次 loss 等指标到 TensorBoard。

- **建议**: 大多数情况下默认值(200)即可。

:::

::: details 各参数详情 — 高级设置 (5 个)
#### `train.learning_rate` — 学习率
决定模型每一步修正幅度的大小。类似自行车把手的灵敏度。

- **提高**: 学习更快但值可能跳动,训练不稳定或发散。
- **降低**: 更稳定但训练非常慢,可能无法达到最优点。
- **建议**: 不确定的话就用默认值(0.0001)。

#### `train.warmup_epochs` — 预热轮数
训练初期将学习率从 0 逐渐提升的区间。类似汽车起步时不急加速而是缓慢提速。

- **建议**: 0~3 轮。如果训练初期不稳定可尝试设为 1~2。

#### `train.freeze_JP_bert` — 冻结 JP BERT 编码器
固定理解文本的部分(BERT)的权重。

- **开启**: 保留 BERT 已学习的日语知识。数据较少时(约 200 句以下)有效防止过拟合,速度和内存也会改善。
- **关闭**: BERT 也一起微调,数据充分时可以学到更自然的发音和韵律。
- **建议**: 200 句以下 → ON,500 句以上 → OFF。

#### `train.freeze_style` — 冻结风格编码器
固定表达说话人说话方式/情绪的风格编码器权重。

- **开启**: 保留现有的风格表现力。数据较少或想保持现有情感表现力时有用。
- **关闭**: 自由学习新说话人的风格特征。
- **建议**: 一般设为 OFF。如果训练结果单调可尝试 ON。

#### `train.freeze_decoder` — 冻结解码器
固定最终生成语音的解码器权重。

- **开启**: 语音生成部分不变,仅训练文本解析部分。训练更快但音质改善幅度受限。
- **关闭**: 解码器也一起训练,可以产出适合该说话人的音质。
- **建议**: 一般设为 OFF。除非特殊情况无需开启。
:::

点击 **开始训练** 后会先显示 **简报面板**。轮数·批大小·总步数·检查点间隔·预计磁盘占用整理在一张表中,检查点间隔可以当场直接修改。

```text
╭──────────────────────────────── tsukuyomi — 训练简报 ────────────────────────────────╮
│    超参数                                                                             │
│      数据集                            65 train / 7 val                               │
│      轮数                                            500                              │
│      批大小                                            2                              │
│      学习率                                        0.0001                              │
│      bfloat16                                         OFF                              │
│                                                                                        │
│    步数 & 检查点                                                                      │
│      每轮步数                                         33                              │
│      总步数                                       16,500                              │
│      记录间隔                                  200 steps                              │
│      检查点间隔                           1000 steps                                  │
│      预计保存次数                                  16次                               │
│      预计磁盘占用             ~3.8GB (16个 × 240MB)                                   │
│                                                                                        │

│      推理模型           data/dataset/tsukuyomi/exports                                │

  ❯ 开始训练
    修改检查点间隔 (当前: 1000 steps)
```

确认后继续,训练开始,TensorBoard 也会自动同时启动。在浏览器中打开控制台打印的 URL 即可实时查看 loss 变化。

::: tip TensorBoard 会自动启动
无需另外输入 `tensorboard --logdir ...`。

训练结束或用 `Ctrl-C` 中断时会自动关闭。
:::

::: details 内部机制 — 训练开始时发生的事
1. **预训练模型自动下载** — 从 `hf://lemondouble/hayakoe/pretrained/<name>` 下载 `G_0.safetensors`, `D_0.safetensors`, `WD_0.safetensors` 到缓存。如果已有 `G_*.pth` 检查点则判断为"恢复模式"并跳过。
2. **训练进程启动** — 通过 `torch.distributed.run` 子进程启动 `train_ms_jp_extra.py`(SBV2 原始移植)。`nproc_per_node` 值直接传递。
3. **TensorBoard 自动运行** — 指向训练目录的 TensorBoard 同时启动。

预计磁盘占用按每个检查点 240MB 固定估算。
:::

## 训练产出

训练完成后数据集目录下会填充两个路径。

- **`training/`** — 训练中间快照(检查点·TensorBoard 日志)。占用磁盘较多,训练后不需要的检查点可以删除。
- **`exports/<model_name>/`** — 作为部署和评估基准的最终模型。下一步([③ 质量报告](/zh/training/quality-check),[④ 部署](/zh/training/publish))读取此目录。

::: details 内部机制 — 目录结构
```
<dataset_root>/
├── training/                          # 训练中间产出
│   ├── G_*.pth, D_*.pth, WD_*.pth    # 检查点 (每 eval_interval)
│   └── events.out.tfevents.*          # TensorBoard 日志
└── exports/<model_name>/              # 最终模型
    ├── *.safetensors
    ├── config.json
    └── style_vectors.npy
```
:::

## 中断与恢复

用 `Ctrl-C` 中断训练后检查点仍保留在 `training/` 下。

用相同数据集再次点击 **开始训练** 时会自动检测到残留检查点并进入恢复模式。此时跳过预训练下载。
