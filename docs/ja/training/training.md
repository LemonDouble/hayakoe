# ② 前処理 & 学習

[① データ準備](/ja/training/data-prep) で作成したデータセットを学習用に加工し、声を学習します。

学習はコンピュータリソースを非常に多く使う長時間の作業です。GUI もリソースを消費するため、UI が必要なデータ前処理パイプラインの後はリソースをほぼ消費しない CLI に分離してあります。

コマンドを覚える必要はありません。CLI を実行した後、メニューに沿って進むだけです。

## CLI 実行

リポジトリルートで引数なしに実行するとメインメニューが表示されます。

```bash
uv run poe cli
```

## 前処理

データ準備で作った `(音声, テキスト)` ペアは、モデルがそのまま学習するには合わない形式です。この元データを **モデルが理解できる数値形式に変換してキャッシュしておくステップ** が前処理です。

データセットを選ぶと現在の前処理状態パネルがまず表示されます。

```text
╭──────────────────────────────── つくよみ — 前処理状態 ────────────────────────────────╮
│    ✓  オーディオファイル    72個                                                          │
│    ○  テキスト前処理      未完了                                                        │
│    ○  BERT 埋め込み        0/72                                                          │
│    ○  スタイルベクトル    0/72                                                          │
│    ○  デフォルトスタイル    未完了                                                        │
╰────────────────────────────────────────────────────────────────────────────────────────╯
次のステップ: 前処理実行でデータを準備してください。

? つくよみ — 何をしますか？
  ❯ 前処理実行（残りのステップ）
    前処理全体再実行
    学習設定編集
    戻る
```

**前処理実行（残りのステップ）** を選択すると、学習に必要な4ステップの前処理が順番に実行されます。

完了判定は自動で行われるため、途中で中断して戻ってきても終わったステップはスキップされます。すべてやり直したい場合は **前処理全体再実行** を使ってください。

::: tip G2P エラーが出た場合
前処理1ステップ目で `text_error.log` が生成された場合、一部の文が音素変換に失敗したことを意味します。

該当行は学習対象から自動的に除外されますが、データが少ない状況ならログを開いて原因を確認してみてください。特殊記号や読めない漢字が原因の場合が多いです。
:::

::: details 内部動作 — 4ステップ詳細
| ステップ | 内容 | 成果物 |
|---|---|---|
| 1. G2P テキスト前処理 | 日本語テキストを発音（音素 + アクセントトーン）に変換。モデルは文字ではなく **発音** を学習します。 | `train.list` / `val.list` が7フィールドに拡張、`text_error.log` |
| 2. BERT 埋め込み | 文脈を DeBERTa 埋め込みで事前計算してキャッシュ。自然な抑揚のために参照する値ですが、学習ステップごとに BERT を実行すると遅いので一度だけ計算します。 | `<wav>.bert.pt` |
| 3. スタイルベクトル | 各音声から話者の声の特徴（トーン・話し方）をベクトルとして抽出（pyannote 埋め込み）。 | `<wav>.npy`（NaN ファイルは自動削除） |
| 4. デフォルトスタイル | 上記ベクトルの平均を話者の「標準スタイル」として保存。 | `exports/<model>/style_vectors.npy`、`config.json` |

完了判定はファイル数のマッチングで行います。
:::

::: tip 総ステップ 2000~4000 が通常の基準
経験上 **総ステップ 2000~4000** の区間で使える音声が出る場合が多いです。エポック数はデータセットサイズとバッチサイズによって変わるため、ブリーフィングパネルの `総ステップ` がこの範囲に入るよう `epochs` を調整してください。

この範囲内でチェックポイントが複数出るよう `eval_interval` を 500~1000 程度に設定すると、後で [品質レポート](/ja/training/quality-check) で最適な時点を選ぶ際に便利です。
:::

各パラメータの意味と推奨値（上げると / 下げると / 推奨）は折りたたんであります。調整が必要なときに展開してください。

::: details パラメータ別詳細 — 基本設定（6個）
#### `train.epochs` — 学習エポック数
全学習データを最初から最後まで何回繰り返し学習するかを設定します。

- **上げると**: モデルがより長く学習し音質が向上する可能性があります。ただし上げすぎると学習データを丸暗記してしまい、新しい文を読むときにかえって不自然になります（過学習）。
- **下げると**: 学習が早く終わりますが、モデルが十分に学習できず音質が低い場合があります。
- **推奨**: 総ステップが 2000~4000 に収まるよう調整してください。`エポック = 目標ステップ × バッチサイズ ÷ 学習データ数` で計算できます。

#### `train.batch_size` — バッチサイズ（GPUあたり）
一度に何個の音声サンプルをまとめて学習するかを設定します。

- **上げると**: 学習速度が速くなり安定しますが、GPU メモリ（VRAM）をより多く使用します。メモリ不足だと `CUDA out of memory` エラーが発生します。
- **下げると**: 少ないメモリでも学習可能ですが、速度が遅くなります。
- **推奨**: RTX 3060 (12GB) → 2~4、RTX 3090/4090 (24GB) → 4~8。

#### `train.nproc_per_node` — GPU 数
学習に使用する GPU の数です。複数の GPU がある場合、データを分けて並列に学習し速度が比例して速くなります（DistributedDataParallel）。

- **デフォルト 1**: 単一 GPU 学習。GPU が1枚ならこの値を変えないでください。
- 実際の搭載数を超えるとエラーが発生します。`nvidia-smi` で確認できます。

#### `train.bf16_run` — bfloat16 混合精度
計算精度を32ビットから16ビットに下げて学習速度を上げる手法です。

- **ONにすると**: 学習速度 ~1.5倍、VRAM 使用量削減。ただし Ampere 以降の GPU（RTX 30xx、40xx、A100 など）が必要です。
- **OFFにすると**: すべての GPU で動作しますがより遅くなります。
- **注意**: RTX 20xx 以下で ON にするとエラーが出たり品質が落ちたりする場合があります。

#### `train.eval_interval` — チェックポイント保存間隔（ステップ）
何ステップごとにチェックポイントを保存するかを設定します。チェックポイントは学習中のモデルのスナップショットで、後で最も良い時点のモデルを選んで使えるようにしてくれます。

- **頻繁に保存**: 細かく比較可能、代わりにディスク使用量が大きくなります（チェックポイントあたり ~240MB）。
- **まれに保存**: ディスクは節約されますが良い時点を見逃す可能性があります。
- **推奨**: 500~1000 ステップ。総ステップ（2000~4000）の区間でチェックポイントが4~8個残るように設定すると品質レポートで比較する際に便利です。

#### `train.log_interval` — TensorBoard ログ間隔（ステップ）
何ステップごとに loss などの指標を TensorBoard に記録するかを設定します。

- **推奨**: ほとんどの場合デフォルト値（200）で十分です。

:::

::: details パラメータ別詳細 — 詳細設定（5個）
#### `train.learning_rate` — 学習率
モデルが1ステップごとにどれだけ大きく修正されるかを決定します。自転車のハンドルの感度に似ています。

- **上げると**: 速く学習しますが値が飛んで学習が不安定になったり発散する場合があります。
- **下げると**: 安定的ですが学習が非常に遅く、最適点に到達できない場合があります。
- **推奨**: よくわからなければデフォルト値（0.0001）をそのまま使ってください。

#### `train.warmup_epochs` — ウォームアップエポック
学習初期に学習率を0から徐々に上げる区間です。車を発進させるとき急加速ではなくゆっくり速度を上げるのと似ています。

- **推奨**: 0~3 エポック。学習が初期に不安定なら1~2に設定してみてください。

#### `train.freeze_JP_bert` — JP BERT エンコーダー凍結
テキストを理解する部分（BERT）の重みを固定します。

- **ONにすると**: BERT が既に学習した日本語の知識を保持します。データが少ないとき（~200文以下）の過学習防止に効果的で、速度とメモリも改善されます。
- **OFFにすると**: BERT も一緒にファインチューニングされ、データが十分なときにより自然な発音と抑揚を学習します。
- **推奨**: 200文以下 → ON、500文以上 → OFF。

#### `train.freeze_style` — スタイルエンコーダー凍結
話者の話し方/感情を表現するスタイルエンコーダーの重みを固定します。

- **ONにすると**: 既存のスタイル表現力を保持します。データが少ない場合や既存の感情表現力を維持したいときに有用です。
- **OFFにすると**: 新しい話者のスタイル特性を自由に学習します。
- **推奨**: 一般的に OFF。学習結果が単調なら ON で試してみてください。

#### `train.freeze_decoder` — デコーダー凍結
最終的な音声を生成するデコーダーの重みを固定します。

- **ONにすると**: 音声生成部分は変更されず、テキスト解釈部分のみが学習されます。学習は速くなりますが音質改善の幅が制限されます。
- **OFFにすると**: デコーダーも一緒に学習し、その話者に合った音質を出せます。
- **推奨**: 一般的に OFF。特殊な場合でなければ OFF のままで問題ありません。
:::

## 学習開始

**学習開始** を押すと **ブリーフィングパネル** がまず表示されます。エポック・バッチサイズ・総ステップ・チェックポイント間隔・予想ディスク使用量がひとつの表にまとめて出て、チェックポイント間隔だけその場ですぐに変更できます。

```text
╭──────────────────────────────── つくよみ — 学習ブリーフィング ────────────────────────────────╮
│    ハイパーパラメータ                                                                      │
│      データセット                            65 train / 7 val                              │
│      エポック                                           500                              │
│      バッチサイズ                                          2                              │
│      学習率                                        0.0001                              │
│      bfloat16                                         OFF                              │
│                                                                                        │
│    ステップ & チェックポイント                                                           │
│      エポックあたりステップ                             33                              │
│      総ステップ                                       16,500                              │
│      ログ間隔                                  200 steps                              │
│      チェックポイント間隔                           1000 steps                              │
│      予想保存回数                                  16回                              │
│      予想ディスク使用量             ~3.8GB (16個 × 240MB)                              │
│                                                                                        │
│    保存パス                                                                           │
│      チェックポイント            data/dataset/つくよみ/training                              │
│      推論モデル              data/dataset/つくよみ/exports                              │
╰────────────────────────────────────────────────────────────────────────────────────────╯

? 続行
  ❯ 学習開始
    チェックポイント間隔の変更（現在: 1000 steps）
    キャンセル
```

確認して続行すると学習が開始され、TensorBoard も自動的に一緒に起動します。コンソールに表示された URL をブラウザで開くと loss の変化をリアルタイムで確認できます。

::: tip TensorBoard は自動で起動します
別途 `tensorboard --logdir ...` を打つ必要はありません。

学習が終了するか `Ctrl-C` で中断すると自動的に閉じます。
:::

::: details 内部動作 — 学習開始時に起こること
1. **事前学習モデルの自動ダウンロード** — `hf://lemondouble/hayakoe/pretrained/<name>` から `G_0.safetensors`、`D_0.safetensors`、`WD_0.safetensors` をキャッシュにダウンロードします。既に `G_*.pth` チェックポイントがあれば「再開モード」と判断してスキップします。
2. **学習プロセスの起動** — `torch.distributed.run` サブプロセスで `train_ms_jp_extra.py`（SBV2 オリジナルのポーティング）を起動します。`nproc_per_node` の値がそのまま渡されます。
3. **TensorBoard の自動実行** — 学習ディレクトリを参照する TensorBoard が一緒に起動します。

予想ディスク使用量はチェックポイントあたり 240MB 固定推定です。
:::

## 学習成果物

学習が終了するとデータセットディレクトリ以下の2つのパスが作成されます。

- **`training/`** — 学習途中のスナップショット（チェックポイント・TensorBoard ログ）。ディスクを多く消費するため、学習後に不要なチェックポイントは削除しても構いません。
- **`exports/<model_name>/`** — デプロイと評価の基準となる最終モデル。次のステップ（[③ 品質レポート](/ja/training/quality-check)、[④ デプロイ](/ja/training/publish)）がこのディレクトリを読み取ります。

::: details 内部動作 — ディレクトリ構造
```
<dataset_root>/
├── training/                          # 学習途中の成果物
│   ├── G_*.pth, D_*.pth, WD_*.pth    # チェックポイント (eval_interval ごと)
│   └── events.out.tfevents.*          # TensorBoard ログ
└── exports/<model_name>/              # 最終モデル
    ├── *.safetensors
    ├── config.json
    └── style_vectors.npy
```
:::

## 中断と再開

学習を `Ctrl-C` で中断してもチェックポイントは `training/` 以下に残ります。

同じデータセットで再度 **学習開始** を押すと、残っているチェックポイントを自動的に検出して再開モードに入ります。事前学習のダウンロードはこの場合スキップされます。
